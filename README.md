# Provenance Guard

A Flask backend API that classifies submitted text as AI-generated or human-written, returning a structured JSON response with a confidence score, individual signal scores, and a plain-language attribution label.

**Port:** 5001  
**Endpoints:** `POST /submit`, `POST /appeal`, `GET /log`

## 1. Project Overview

Provenance Guard addresses a growing need on creative writing platforms: readers and platforms need a way to know whether posted content was written by a person or generated by an AI tool, without pretending that the result is perfectly certain.

- **Transparency over verdicts.** The system returns plain-language labels that describe evidence, not verdicts. It acknowledges uncertainty explicitly rather than forcing a binary outcome.
- **False positives cost more than false negatives.** Mislabeling a human's original work as AI-generated harms creators. The system is intentionally conservative: it requires high confidence before applying an AI label.

The system does not automate consequences. It classifies, logs, and surfaces information for human review.

## 2. Architecture Overview

A submitted text travels through five components before a response is returned.

A POST request to `/submit` arrives at `app.py`, which validates that both `text` and `creator_id` are present and generates a UUID for the submission. The text is then passed simultaneously to two signal pipelines: a Groq-based LLM classifier and a stylometric heuristic engine. Their outputs are combined into a single confidence score, which is then translated into a transparency label. Finally, the submission is written to the SQLite audit log and returned as JSON.

Appeals arrive at `POST /appeal`. `app.py` validates the request, `models/audit.py` confirms the `content_id` exists, and `models/appeal.py` updates the row's status to `under_review` and records the creator's reasoning.

## 3. Detection Signals

### Signal 1: Groq LLM Classification (`pipeline/groq_signal.py`)

**What it measures:** Whether a piece of writing reads as AI-generated based on holistic semantic and stylistic patterns.

**Weight:** 0.6 — semantic holistic judgment is more reliable across content types than structural statistics.

### Signal 2: Stylometric Heuristics (`pipeline/stylometric.py`)

**What it measures:** Three structural and statistical properties of the text that differ systematically between human and AI writing, independent of the text's meaning.

**Weight:** 0.4 — supplementary structural evidence, useful precisely because it is independent of the LLM signal.

## 12. Running the Project

### Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Environment

Create a file named `groq_key.env` in the project root with:

```env
GROQ_API_KEY=your_groq_api_key_here
```

This file is gitignored and must not be committed.

### Start the App

```bash
python3 app.py
```

The server runs on port **5001**.
